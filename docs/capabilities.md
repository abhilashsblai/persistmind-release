# Capabilities

Status is release-specific. "Implemented" means code exists; "qualified" means
the installed artifact passed evidence for the current boundary.

| Capability | `0.2.1.dev4` status |
| --- | --- |
| CLI identity, status, and doctor | Qualified internal preview |
| Local repository indexing | Qualified internal preview |
| Lexical source search | Qualified internal preview |
| Deterministic context packs | Qualified internal preview |
| Task, plan, checkpoint, verification, outcome | Qualified internal preview |
| Local approved memory and recall | Qualified internal preview |
| Local audit operations | Qualified internal preview |
| Storage integrity, backup, and restore | Qualified internal preview |
| Read-only stdio MCP | Qualified policy boundary |
| Loopback HTTP implementation | Not part of the supported preview workflow |
| Team/server mode | Disabled |
| Remote authorization and writes | Disabled |
| Writable MCP | Disabled |
| Cross-repository authority | Unsupported |
| Cognitive improvement | Experimental; not production-authoritative |
| Anticipation | Experimental; not production-authoritative |
| Automatic self-improvement adoption | Disabled |
| Autonomous source modification | Disabled |

Advanced capabilities require independent sample-size, safety, rollback, and
usefulness gates before promotion.
