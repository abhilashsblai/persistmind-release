# CTX Layer 0.2.0a14

Released July 12, 2026.

This corrective preview release makes legacy-project migration part of normal
GitHub wheel updates for every installation, including projects where the
bootstrap package was installed before project storage was upgraded.

## Corrections

- Project storage state is evaluated before `already_current` is returned.
- A legacy `workspace.db` without an active split catalog triggers the
  candidate-owned project upgrade even when package versions are equal.
- Unknown FTS and sqlite-vec virtual tables are classified as rebuildable
  source indexes. Version-specific tables such as `vec_chunks_128` no longer
  block migration.
- Virtual-table shadow data is not treated as durable project learning.
- Knowledge, learning, task, activity, audit, and policy tables still require
  declared ownership and pass row-count/content-hash verification.
- Original monolith, verified backup, restore drill, and rollback evidence are
  retained.

## Install

```powershell
python -m pip install --upgrade "https://github.com/abhilashsblai/ctxlayer-release/releases/download/v0.2.0a14/ctxlayer-0.2.0a14-py3-none-any.whl"
```

## Update Existing Projects

Projects already running `0.2.0a13` do not need a manual version override:

```text
cd <project>
ctxlayer update --check
ctxlayer update --dry-run
ctxlayer update
```

The preview resolver selects `0.2.0a14` because it is newer than `0.2.0a13`.
Future releases follow the same channel/version mechanism, so manual
`--version` selection is not required for normal updates.
