# CTX Layer 0.2.0a15

Released July 12, 2026.

This corrective bootstrap release fixes candidate installation from CTX
Layer's verified, content-addressed wheel cache.

## Correction

The updater caches a verified wheel as `<sha256>.whl`. Pip correctly rejects
that digest-only basename because it is not a standards-compliant wheel
filename. The updater now copies the already verified bytes into the temporary
candidate slot using the canonical filename declared by the signed manifest,
for example:

```text
ctxlayer-0.2.0a15-py3-none-any.whl
```

The staged copy is removed after candidate installation. The immutable
content-addressed cache, signed manifest verification, digest checks, and
rollback behavior remain unchanged.

## One-Time Bootstrap Repair

Because `0.2.0a14` fails before a newer candidate can start, install this wheel
once:

```powershell
python -m pip install --upgrade "https://github.com/abhilashsblai/ctxlayer-release/releases/download/v0.2.0a15/ctxlayer-0.2.0a15-py3-none-any.whl"
```

Then update each project normally:

```text
ctxlayer update --check
ctxlayer update --dry-run
ctxlayer update
```

Updates after `0.2.0a15` use the corrected bootstrap automatically and do not
require manual wheel installation or `--version` selection.
