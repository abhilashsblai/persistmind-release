# Core Local Profile

Core Local is the supported product boundary represented by the locked
`windows-internal-preview` runtime profile in `0.2.1.dev17`.

## Included

- one local Git repository;
- local filesystem state;
- CLI-based indexing, search, packs, workflow, memory, audit, and recovery;
- local human-reviewed mutations;
- read-only stdio MCP;
- manual approval and review; and
- local storage; backup and recovery qualification remains pending for promotion.

## Excluded

- remote/team service operation;
- non-loopback authority expansion;
- remote authorization and mutation brokers;
- writable MCP;
- automatic updater use for unsigned artifacts;
- autonomous source modification;
- automatic self-improvement adoption;
- cross-repository authority; and
- production-authoritative experimental intelligence.

Core Local describes authority and support, not merely where a process runs. A
caller-provided string, prompt, or actor name cannot expand it.
