# PersistMind 0.2.1.dev17 - Internal Windows Preview

Released July 17, 2026 for restricted local testing.

## Summary

This release fixes a workflow-continuity defect reported from FoxFlow after the
dev16 update. Plans created through the documented manual lifecycle were accepted,
but `workflow next` expected recommendation state written only by `workflow start`.

## Fixed

- Split `workflow next` now recovers missing recommendations for sessions created
  through `task start`, pack binding, and plan creation.
- Recovery uses the authoritative task objective, current bound pack, accepted
  plan paths and capabilities, and persisted impact evidence.
- The recovered snapshot records its provenance and becomes the durable source
  for subsequent calls.
- A missing recommendation is no longer mislabeled as an internal error.

## Verification

The reported FoxFlow task reproduced the failure under dev16. Running the
corrected source recovered its split snapshot, after which the installed dev16
runtime returned the correct bound pack, passing plan/checkpoint/preflight gates,
and the pending diff-check action. Focused workflow/split tests passed, followed
by an isolated Windows 11/Python 3.12 installed-wheel lifecycle qualification.
