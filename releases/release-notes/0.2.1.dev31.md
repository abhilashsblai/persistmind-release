## Corrected behavior

- Task State routing follows the runtime mode: read-only workflow inspection
  resolves readers only, while mutations explicitly require a writer.
- Task show, activity list/show, pack manifest/reconstruction, and read-only MCP
  inspection no longer fail with `read-only storage runtime cannot resolve a
  writer: task_state`.
- `check-diff` and pack-aware impact use a writable Pack service because they
  persist verification evidence; source-impact lookup remains read-only.
- `setup` and `setup auto` dispatch directly to the setup service and correctly
  regenerate hooks, project instructions, and MCP configuration.
- FoxFlow project and global Codex surfaces point at one canonical installer-
  managed runtime, allowing later exact releases to replace that runtime
  without leaving stale source-tree or development-build launchers behind.

## Qualification

Twenty focused tests passed across setup routing, read-only storage routing,
authority integration, and Windows preview behavior. The exact packaged wheel
also passed fresh Windows 11 Pro/Python 3.13 installation, identity, doctor,
indexing, search, context pack, workflow, 92-tool read-only MCP registration,
encrypted backup, staged restore, safe uninstall, and the root PowerShell
installer's `-VerifyOnly` check.

The exact installed dev31 runtime was then deployed into FoxFlow. Task lookup,
MCP task lookup, manifest inspection, activity listing, and a scoped
`check-diff` completed without the reported writer-routing error. A full
FoxFlow working-tree check correctly reaches its governance release gate because
the repository currently contains changes outside the active plan; that is a
scope result, not a storage failure.

The test host's Windows Credential Manager was saturated by pre-existing test
credentials. Backup and restore were exercised with PersistMind's explicit
development key-provider switch; the production keyring allocation path was not
observed in this run.

## Restrictions

Use only disposable or non-critical local repositories. Manual review is
mandatory. Windows 10, Python 3.11/3.12, Linux, and macOS remain unqualified for
this candidate. Qualification was intentionally focused on the corrected and
installed-artifact paths; no full historical regression-suite claim is made.

Remote writes, writable MCP, autonomous source modification, automatic
self-improvement adoption, team/server mode, cross-repository authority, and
production-authoritative experimental intelligence remain unsupported. The
trusted updater must not install this unsigned preview.
