# PersistMind 0.2.1.dev18 - Internal Windows Preview

Released July 17, 2026 for restricted local testing.

## Summary

This release fixes a hook-policy freshness defect reported from FoxFlow. A
successful preflight could be rejected moments later because PersistMind treated
any task database or WAL write as a write-scope revision, and read-only workflow
inspection could overwrite the green policy snapshot.

## Fixed

- Source freshness uses the published source generation rather than database
  file timestamps.
- Write-scope freshness uses the bound task, pack, plan status, and plan revision.
- Trust freshness uses the policy domain revision.
- Unrelated task/checkpoint telemetry does not revoke a valid preflight.
- Read-only workflow inspection leaves the authoritative preflight snapshot intact.
- Plan amendments, pack rebinding, source updates, trust changes, and expiry remain
  fail-closed transitions.

## Verification

Focused hook, runtime-operation, packaging, and release checks passed. The exact

was then installed into FoxFlow, setup regenerated the project surfaces, the
