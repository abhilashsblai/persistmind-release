## Candidate changes

- Adds bounded, query-aware context snippets to Codex integration.
- Tightens read-only hook gating so read-only operations do not require a
  write-authority path.
- Migrates the legacy pytest suite to the next-generation authority and schema
  architecture, including updated fixtures, helpers, and storage contracts.
- Repairs authority schema upgrade lifecycle behavior and preview install
  verification/rollback.
- Includes dev38-dev40 next-generation authority, workflow, reporting, and
  schema-head changes.

## Qualification evidence

The exact wheel passed:

- exact source-commit and version identity;
- isolated offline installation on Windows 11 with Python 3.12;
- import from the installed environment rather than the source checkout;
- required operation catalog and storage manifest packaging;
- CLI help and package-resource checks; and
- checksum-verifying installer `-VerifyOnly`.

The complete source suite collected 565 tests and passed locally in serial and
four-worker modes, with four environment-dependent skips. The cross-platform
GitHub matrix was blocked before any runner started because of the account
billing/spending limit; this is an infrastructure limitation, not a test result.

Full APSW-backed install, indexing, search, context-pack, workflow, backup, and
restore qualification is not claimed. Windows Application Control on the
release host rejected the pinned APSW native module. Removing download-zone
metadata did not override that enterprise policy.

## Installation

Follow the [Windows installation guide](../../docs/windows-installation.md).
Install only into a new isolated environment and use only a disposable or
non-critical repository. Stop if APSW is blocked or `doctor` is degraded. Do not
use `persistmind update` and do not bypass organizational security policy.
